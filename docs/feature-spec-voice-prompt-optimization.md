# 시스템 프롬프트 음성 최적화 기능정의서

## 1. 개요
- AI 콜봇 시스템 프롬프트를 음성 통화에 최적화하여 응답 길이를 줄이고 대화체로 전환
- 핵심 가치: 전화 통화에서 자연스럽고 빠른 고객 응대

## 2. 배경 및 목적
- **문제**: 현재 시스템 프롬프트에 음성 채널 특화 지침이 없어 LLM이 장문의 텍스트형 응답을 생성
- **As-Is**: "요금 조회" 질문에 5~6문장 이상의 상세 설명 응답 → 전화 통화에서 답답함
- **To-Be**: 1~2문장의 간결한 구어체 응답 → 자연스러운 전화 상담 경험
- **비즈니스 임팩트**: 평균 통화 시간 단축, 고객 만족도 향상, 이탈률 감소 (기술 완료 기준으로 측정하며, 비즈니스 지표는 라이브 운영 후 후속 측정 예정)

## 3. 용어 정의

| 용어 | 정의 |
|------|------|
| 음성 응답 | TTS를 통해 사용자에게 전달되는 AI 응답 |
| 턴(Turn) | 사용자 발화 1회 + AI 응답 1회의 한 사이클 |
| 시스템 프롬프트 | LLM에게 역할/제약을 지시하는 사전 프롬프트 |
| 응답 글자 수 | 응답 텍스트의 글자(character) 수. 코드 변수명 `max_syllables`에 대응. 현재 기본값 150자. |

## 4. 사용자 스토리

- **US-001**: 고객으로서, 요금 조회 시 핵심 정보(금액)만 빠르게 듣고 싶다
- **US-002**: 고객으로서, AI가 전화 상담원처럼 자연스러운 말투로 응대받고 싶다
- **US-003**: 고객으로서, 추가 정보가 필요하면 내가 물어봐서 듣고 싶다 (먼저 다 말하지 말고)

## 5. 기능 요구사항

| ID | 요구사항 | 우선순위 | 관련 US |
|----|---------|---------|---------|

## 7. 기술 설계

### 변경 대상 파일
1. `llm_engine/prompt_loader.py` — 인텐트별 프롬프트 텍스트 수정
2. `llm_engine/llm_engine.py` — `_build_system_prompt()` 수정, `_MAX_SYLLABLES_DEFAULT` 150→80 조정
3. 기존 테스트 파일 (`llm_engine/tests/test_*.py` 등) — 프롬프트 변경에 따른 assertion 업데이트

### 프롬프트 설계 원칙
- **간결성**: 핵심 정보 1~2문장 + 대화 유도 1문장(선택). 부연 설명 금지.
- **구어체**: "~입니다" → "~이에요", "~해 드리겠습니다" → "~해 드릴게요"
- **핵심 우선**: 숫자/금액은 바로 말하기. "현재 요금은 65,000원이에요."
- **대화 유도**: 추가 안내가 필요하면 "더 궁금하신 거 있으세요?"로 마무리
- **서식 금지**: 마크다운 문법(불릿 기호 `- *`, 번호 기호 `1.`, 헤더 `#` 등) 사용 금지. 음성 나열은 "첫 번째는 ~, 두 번째는 ~" 형태로 허용.

### 인텐트별 필수 포함 정보
| 인텐트 | 필수 포함 | 예시 응답 (80자 이내) |
|--------|----------|---------------------|
| BILLING_INQUIRY | 금액 | "이번 달 요금은 65,000원이에요. 더 궁금하신 거 있으세요?" (33자) |
| PAYMENT_CHECK | 납부 상태, 금액 | "최근 납부는 3월 15일에 65,000원 완료됐어요." (28자) |
| DATA_USAGE_INQUIRY | 잔여량 | "데이터 3.2GB 남아 있어요. 초기화일은 4월 1일이에요." (31자) |
| PLAN_INQUIRY | 요금제명, 월 요금 | "첫 번째는 5G 스탠다드 월 55,000원, 두 번째는 5G 프리미어 월 75,000원이에요." (48자) |
| PLAN_CHANGE | 변경 전후 정보 | "5G 프리미어로 변경하면 월 75,000원이에요. 진행할까요?" (33자) |
