# Phase E: NLU 고도화 기능정의서

## 1. 개요
- 콜봇 NLU(Natural Language Understanding) 엔진의 구어체 처리 능력과 다단계 플로우 중 인텐트 전환 기능을 강화한다.
- 핵심 가치: 자연스러운 한국어 구어체 발화를 정확히 분류하고, 진행 중인 플로우에서 사용자가 의도를 변경할 때 자연스럽게 전환할 수 있게 한다.

## 2. 배경 및 목적
- **해결하려는 문제**: QA #5 — "요금 좀 알려줘", "그거 얼마야" 같은 구어체 발화가 UNCLASSIFIED로 분류됨. QA #6 — 요금 조회 플로우 중 "아 그거 말고 부가서비스 해지해줘" 시 인텐트 전환 불가.
- **As-Is**: 키워드 기반 정확 매칭 (`_KEYWORD_RULES`). 구어체, 줄임말, 조사 변형에 취약. 다단계 플로우 중 새 인텐트 발화 시 무시되거나 오류.
- **To-Be**: 패턴 확장 + 유사 표현 매핑으로 구어체 커버리지 향상. 다단계 플로우 중 인텐트 전환 지원.
- **비즈니스 임팩트**: NLU 인식률 향상 → UNCLASSIFIED 비율 감소 → 상담사 연결 감소 → 운영 비용 절감

## 3. 용어 정의

| 용어 | 정의 |
|------|------|
| 구어체 패턴 | 일상 대화에서 사용하는 비공식 표현 (예: "요금 좀", "얼마야", "해줘") |
| 인텐트 전환 | 다단계 플로우 진행 중 사용자가 다른 인텐트를 발화하여 플로우를 변경하는 것 |
| 다단계 플로우 | `pending_intent`가 설정된 상태에서 2턴 이상 진행되는 업무 (요금제 변경, 부가서비스 해지 등) |
| 패턴 그룹 | 하나의 인텐트에 매핑되는 키워드/정규식 패턴의 묶음 |
| 전환 확인 | 인텐트 전환 시 사용자에게 기존 플로우 취소를 확인하는 프로세스 |

## 4. 사용자 스토리

- **US-001**: As a 고객, I want 자연스러운 말투로 요금을 물어볼 수 있다, So that 키워드를 정확히 기억하지 않아도 서비스를 이용할 수 있다.
- **US-002**: As a 고객, I want "그거 얼마야", "요금 좀 알려줘" 같은 구어체로 말해도 인식된다, So that 로봇과 대화하는 느낌이 줄어든다.
- **US-003**: As a 고객, I want 요금제 변경 중 "아 그거 말고 부가서비스 해지해줘"라고 말하면 전환된다, So that 처음부터 다시 시작하지 않아도 된다.
- **US-004**: As a 고객, I want 인텐트 전환 시 기존 플로우 취소 확인을 받는다, So that 실수로 진행 중인 작업이 취소되지 않는다.

## 7. 기술 설계

### 아키텍처 개요
- `MockIntentClassifier`의 `_KEYWORD_RULES`를 정규식 기반 `_PATTERN_RULES`로 확장
- `TurnPipeline.process()`에서 `pending_intent` 체크 전에 NLU 분류 실행
- 새 인텐트 감지 시 `_handle_intent_switch()` 메서드로 전환 확인 처리

### 주요 컴포넌트
1. **패턴 매칭 로직**: `MockIntentClassifier` 내부 `_match_primary_intent()` 확장 (정규식 기반, 별도 클래스 불필요)
2. **인텐트 전환 로직**: `TurnPipeline._handle_intent_switch()` 메서드 (별도 클래스 불필요)
3. **패턴 정의**: `nlu/patterns.py` (인텐트별 정규식 패턴 그룹)

### 기술 스택
- Python 3.9, `re` 모듈 (정규식)
- 기존 `MockIntentClassifier` 확장 (새 클래스 불필요)

## 8. 데이터 모델
